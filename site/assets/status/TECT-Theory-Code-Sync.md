## 3. Version-tracking policy (effective 2026-04-15)

Every run directory produced by any TECT driver MUST carry a
`tect_version_manifest.json` emitted by `PDE/tect_version_manifest.py`.
The manifest records:

- `theory_version` вЂ” e.g. `Math37-Step5-2026-04-15`
- `code_git_sha` (if git is available) and SHA-256 fingerprints of the
  four canonical source files (solver, backend, extractor, Bloch
  linearizer)
- `theory_invariants_closed` and `theory_invariants_open` вЂ” copied from
  the module constants, so a run can always be audited against the
  theoretical status at time of execution
- `params`, `run_label`, UTC timestamp, host

Results without a manifest are to be treated as unversioned and must not
be cited in papers. A retroactive manifest may be added for archived
runs, but must carry `schema_version: "1.0-retro"` and a `notes` field
explaining the reconstruction.
