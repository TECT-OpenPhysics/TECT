# TECT public website

Static site exposing the current TECT theory tag, code inventory, numerical
results, math summaries, papers, and full change history. No build step — this
is hand-written HTML with MathJax rendered in-browser from a CDN.

## Layout

Navigation order (2026-04-21 publish-readiness revision, rev 2): Overview → Theory → Notes → Code → Results → Papers → History → Records.

Rev 1 → rev 2 (2026-04-21, later): swapped Results ↔ Code so Code precedes the Results it produces, and renamed "Math notes" → "Notes" (shorter label; filename `math-notes.html` and id `math-notes` unchanged to preserve URLs).

### History vs Records

Both concern past work but along orthogonal axes:

- **History** is indexed by *date* — answers "what happened on day *t*?". Every substantive event (proof, retraction, solver patch, policy change) appears exactly once as a dated entry in the milestone timeline and/or per-tag changelog.
- **Records** is indexed by *claim* — answers "is claim *X* currently proved, failed, or open?". Entries carry stable tags: `F-…` (failure/retraction), `R-…` (numerical result), `D-…` (dead-end), `Q-…` (open question). The four ledgers (Proved / Failed / Open / Index) are append-only.

A single event typically writes to both: one dated row in History, one status transition in Records.

## Independence / co-management rule

The website is **independent of the source tree** in content but **linked back**
for canonical references:

- Pages under `Website/math/` are **editorial summaries** hand-written for
  public consumption. They do not auto-generate from the source notes. They are
  allowed to lag the raw notes until a reviewer has signed off on the public
  wording.
- Each summary page ends with a **Sources** table linking to the raw
  `docs/math/TECT-Math<NN>.tex.txt` (or relevant supplement). This is the only
  load-bearing link from the public website into the source tree.
- `data/version_index.json` is **generated** from `CHANGELOG.md`,
  `PDE/stamp_version_headers.py`, and `docs/status/TECT-Theory-Code-Sync.md` via
  `tools/build_version_index.py`. Do not edit by hand.

The website folder is therefore self-contained for deployment (a static drop)
while remaining synchronised with the canonical source by convention.

## Maintenance

Canonical source → website propagation:

| When you edit… | Re-run / re-check… |
|---|---|
| `CHANGELOG.md` | `python tools/build_version_index.py`; update `Website/changelog.html` entry. |
| `PDE/stamp_version_headers.py` `MODULE_VERSIONS` | `python tools/build_version_index.py`; spot-check `Website/code.html`. |
| `docs/status/TECT-Theory-Code-Sync.md` | `python tools/build_version_index.py`; spot-check `Website/results.html`. |
| `docs/math/TECT-Math<NN>.tex.txt` | Review the affected `Website/math/project-<roman>-*.html` summary and revise if the result it summarises changed. |
| Theory tag bump | Bump headline on `Website/index.html` + `theory.html` + add `Website/changelog.html` entry + prepend new milestone to `Website/timeline.html` and `Website/data/timeline.json`. |
| **Any page content edit** | Update the `<p class="timestamp">Last updated: YYYY-MM-DD</p>` line in that page's footer. Every page in the site carries its own timestamp. |

## Deploy

The `Website/` folder is deployable as-is:

- **GitHub Pages**: set the repository Pages source to `Website/` on the branch of your choice. Tags: 1 commit == 1 deploy.
- **Netlify / Vercel**: publish directory `Website/`, no build command.
- **S3 / CloudFront**: `aws s3 sync Website/ s3://<bucket>/ --delete`.

No local build step is required. MathJax is loaded from the jsDelivr CDN.

## Browser support

Target: modern Chromium / Firefox / Safari. No IE fallback. The CSS uses CSS
custom properties; the JS is MathJax only.

## Accessibility

Tables use `<th>` headers. Navigation uses semantic `<nav>`. Math is rendered
as SVG by MathJax for screen-reader compatibility via the MathJax assistive
tools. Colour contrast in `assets/style.css` is checked against WCAG AA.
