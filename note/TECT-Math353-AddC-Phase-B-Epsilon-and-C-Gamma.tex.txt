\section{Math353-AddA C-$\beta$ correction (documentation)}

The original Math353-AddA \S ``C-$\beta$ scope'' specified that the operator authors a 7-section root \texttt{README.md} per CLAUDE.md \S 9 manuscript discipline. This was incorrect on two counts:

\begin{enumerate}[leftmargin=2em]
\item \textbf{The auto-generation pipeline already exists}: \texttt{Codes/tools/github\_sync\_curate.py::render\_readme()} (line 345) emits \texttt{Github/README.md} on every snapshot run from canonical sources (\texttt{Docs/status/TOE-FACT-SHEET.md}, \texttt{CHANGELOG.md}, etc.) using the 7-section template. The operator does NOT write the README; the operator keeps canonical sources accurate.
\item \textbf{There is no canonical-root \texttt{README.md} target}: only \texttt{Github/README.md} (auto, in the publish-side mirror). Adding a hand-authored canonical-root \texttt{README.md} would conflict with the publish-side auto-generation and is structurally unnecessary.
\end{enumerate}

\textbf{Corrected understanding}:
\begin{itemize}[leftmargin=2em]
\item Content updates: edit canonical sources (TOE-FACT-SHEET, Math notes, CHANGELOG); next snapshot regenerates README. CLAUDE.md \S 9 \emph{does not} bind the auto-generated artefact.
\item Template structure changes: edit \texttt{render\_readme()} itself in \texttt{github\_sync\_curate.py}. CLAUDE.md \S 9 \emph{does} bind any prose strings inside \texttt{render\_readme()} that would constitute manuscript prose.
\item Root \texttt{README.md} is unnecessary; if added by an autonomous agent in error, it should be removed and the auto-generated \texttt{Github/README.md} re-asserted as canonical.
\end{itemize}

The Math353-AddA C-$\beta$ subsection has been edited in place to insert a correction paragraph at the top, retaining the original (incorrect) text below for audit trail. ROADMAP v2 \S 3 C-$\beta$ row updated to ``AUTO-MANAGED'' status.

\section{B-$\epsilon$ implementation}

\subsection{Design}

A new check function \texttt{check\_rewritten\_paths\_resolve()} added to \texttt{Codes/tools/verify\_website.py}, wired into the main check sequence as step 9/9 ``Mirror-tree path resolution''. The check:

\begin{enumerate}[leftmargin=2em]
\item Reads \texttt{Codes/config/mirror.json}; returns immediately if file missing, schema $<$ 2, or \texttt{v0\_compat\_disable\_renames=true} (current default). This makes the check zero-cost in v0\_compat mode.
\item When \texttt{v0\_compat=false} (post-B-$\delta$ cutover):
\begin{itemize}[leftmargin=2em]
\item Walks every \texttt{*.md}, \texttt{*.html}, \texttt{*.tex.txt} file under \texttt{Github/}.
\item Extracts citations via two regex patterns: \texttt{[text](path)} Markdown link and \texttt{href=}/\texttt{src=}/\texttt{data-src=} HTML attribute.
\item Skips external (\texttt{http://}, \texttt{https://}, \texttt{mailto:}, \texttt{data:}) and pure-fragment (\texttt{\#anchor}) cites.
\item Resolves each remaining cite against either the file's parent directory (relative) or \texttt{Github/} root (absolute-ish leading-slash form).
\item Emits an \texttt{[mirror-resolve]} error per non-resolving cite (capped at first 20 with summary count, to keep error output bounded).
\end{itemize}
\end{enumerate}

\subsection{Verification}

\begin{itemize}[leftmargin=2em]
\item AST parse on patched \texttt{verify\_website.py}: PASS.
\item \texttt{safe\_write.py --verify}: no defects.
\item Run with default toggle (\texttt{v0\_compat\_disable\_renames=true}): the new step 9/9 prints its label and immediately returns; \texttt{verify\_website.py} as a whole exits 0/0.
\item Activation (when toggle flipped): tested logically; will be empirically verified at B-$\delta$ time.
\end{itemize}

\section{C-$\gamma$ implementation (narrative softening sweep)}

\subsection{Design}

Operator script applies \texttt{Codes/config/sweep\_rules.json} v0 \texttt{narrative\_softening\_rules.lattice\_to\_condensate} (4 rules) and \texttt{bcc\_softening\_for\_public\_text} (2 rules) to \texttt{Website/data/\_narrative/*.md} with math-mode masking:

\begin{enumerate}[leftmargin=2em]
\item Compile two regex patterns: \texttt{\$\$...\$\$} block (DOTALL) and \texttt{\$...\$} inline (single-line, non-empty).
\item Replace each math region with sentinel \texttt{\_\_MATH\_BLOCK\_$n$\_\_} or \texttt{\_\_MATH\_INLINE\_$n$\_\_}; collect originals in a list.
\item Apply each \texttt{from} $\to$ \texttt{to} substitution rule in order on the masked text.
\item Restore math regions by inverse substitution (sentinel $\to$ original).
\end{enumerate}

\subsection{Result}

Files scanned: 27 (\texttt{Website/data/\_narrative/*.md}).\\
Files modified: 1 (\texttt{Website/data/\_narrative/papers\_index.md}, 1 substitution).\\
Math regions masked: 29 in the modified file (no LaTeX corruption).\\

The single substitution is consistent with prior-session expectations: most narrative was softened in the post-Math348/350 reframing rounds. The sweep is now codified and reproducible (next narrative addition will be screened automatically).

\paragraph{Objection $\beta$.} ``C-$\gamma$ produced only 1 substitution; the narrative-softening sweep claim of utility is overstated.''\\
\textbf{Disposition:} VALID-WITH-MITIGATION-IS-AUTOMATIC. Counter: the sweep's value is forward-looking, not backward-looking.
