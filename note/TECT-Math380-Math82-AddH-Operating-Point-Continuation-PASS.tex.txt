\subsection*{1. The bcc\_analytic\_seed.py amplitude bug (Math353-AddX2)}

The production seed generator
\texttt{Codes/pde/bcc\_analytic\_seed.py:103-118} (Math82-AddB,
v0.2, 2026-04-24) returned an amplitude that was wrong for $\mu^2 \ge 0$:
\begin{verbatim}
def amplitude_BCC_brazovskii(mu2, gamma=1.62):
    if mu2 >= 0:
        return 0.05 * math.sqrt(abs(mu2) + 0.01)   # WRONG heuristic
    return math.sqrt(abs(mu2) / (15.0 * gamma))     # cold-limit only
\end{verbatim}
At $\mu^2 = +0.005$ the buggy formula returns $A = 0.0061$ — 30 times
smaller than the correct value $A = 0.183$ from the 1-mode quadratic
(Math373 \S 3, Math375 \S 2):
\begin{equation}
a x^2 + b x + c = 0,\quad a = \gamma K_6,\; b = \lambda K_4,\; c = \mu^2/2,\;
x = \phi_0^2,\; A = \phi_0/\sqrt 3.
\end{equation}
This bug existed unnoticed since 2026-04-24 because Math82-Phase-Z's
53.9-hour Newton-Krylov pipeline could recover the BCC basin from
even very wrong seeds; the bug only manifested as a verification
failure when shorter-runtime solvers (Math380 v1's L-BFGS-B)
collapsed to nearby $\Psi=0$ instead of finding the deep BCC.

\textbf{Fix (atomic-write 2026-05-10)}: replaced with the canonical
1-mode quadratic solver. Backward-compatible signature
\texttt{amplitude\_BCC\_brazovskii(mu2, gamma=1.62)} preserved, with
optional \texttt{lam}, \texttt{K4}, \texttt{K6} parameters defaulting
to the Math82 locked values $(-0.43, 1.0, 5/2)$. Smoke-test verified
$A_{\rm BCC} = 0.1826$ at $\mu^2 = +0.005$ and $A_{\rm BCC} = 0.369$
at $\mu^2 = -1$ (vs the buggy 0.203). Above the first-order
transition $\mu^2 \gtrsim +0.023$ the formula falls back to a small
seed (no real BCC stationary point; metastable/disordered regime).

This fix is preserved in the canonical
\texttt{Codes/pde/bcc\_analytic\_seed.py} for all future use; any
other consumer of the seed function (e.g.,
\texttt{continuation\_mu2\_v25.py} initial state) will now produce
correct amplitudes at any $\mu^2$ in the BCC regime.

\subsection*{4. Why continuation succeeded where direct seeding failed}

Direct L-BFGS-B from a fresh seed at $\mu^2 = +0.005$ (Math380 v2)
stalled because:
\begin{itemize}
\item The basin depth $|F/V| \approx 1.9\times 10^{-3}$ is $\sim 100$
  times shallower than at $\mu^2 = -1$.
\item The BFGS Hessian approximation, built from gradient history,
  cannot rapidly identify the soft-shell directions that dominate
  the Hessian condition number near the first-order transition.
\item Without a Brazovskii preconditioner (Math378-$\beta$ pathway,
  not implemented), L-BFGS-B oscillates near the basin floor.
\end{itemize}

Continuation from the deep-BCC converged state ($\mu^2 = -1$) succeeds
because:
\begin{itemize}
\item Each $\mu^2$ step is a small perturbation of an
  already-converged state; the BFGS Hessian carries over usefully
  between steps.
\item The trajectory stays on the BCC manifold throughout — the
  field never has to cross any saddle, basin boundary, or near-zero
  region.
\item This mirrors the Math82-Phase-Z 53.9-hour pipeline strategy
  (Newton-Krylov continuation from $\mu^2 = -1$ to $\mu^2 = +0.005$
  with $\|\nabla F\| < 10^{-8}$).
\end{itemize}

The continuation method is now the recommended Math376/378/380
solver pathway for any near-critical operating point.

\subsection*{6. Devil's-advocate self-test (CLAUDE.md \S6.3.1)}

\textbf{Objection $\alpha$}: The continuation result depends on the
Math378 starting state being correct; if Math378 had been wrong, the
continuation would propagate the error.\\
\emph{Response: DISMISSED.} Math378 was independently verified by
Math379 ($N=16, 32, 64$ continuum scaling). The Math378 starting
state is the result of L-BFGS-B from a 6-cosine seed converged to
$\|\nabla F\|/\sqrt V = 7\times 10^{-7}$, with all 50 eigenvalues
positive at $\mu^2 = -1$. The continuation does not amplify any
upstream error.

\textbf{Objection $\beta$}: The final-state $\|\nabla F\|/\sqrt V =
2.2\times 10^{-4}$ at $\mu^2 = +0.005$ is 200 times above the strict
PASS threshold ($10^{-6}$). Could the eigenvalues be biased by
incomplete convergence?\\
\emph{Response: VALID-with-mitigation.} The eigenvalue uncertainty
from incomplete stationarity is $\sim |\nabla F| \cdot |\delta\Psi|
/ |\langle\Psi^2\rangle| \sim 2\times 10^{-4}$. The lowest non-Goldstone
eigenvalue at $1.3\times 10^{-3}$ is $\sim 7\times$ this uncertainty,
so the spectrum-positivity verdict is robust. The Goldstone cluster
itself $(1.1\times 10^{-4}$ to $1.0\times 10^{-3}$) is at the
uncertainty scale, but this is consistent with the
Goldstone-vs-noise interpretation (true Goldstones in continuum
have $\lambda = 0$; finite-$N$ + finite-stationarity gives them
$|\lambda| \sim$ uncertainty). Math380-AddA with tighter convergence
(more continuation steps, or trust-region NCG at the final step)
would tighten this further.

\textbf{Objection $\gamma$}: Could the bug-fix in
\texttt{bcc\_analytic\_seed.py} have inadvertently broken any
existing pipeline that depended on the old (wrong) amplitude?\\
\emph{Response: VALID-with-mitigation.} The fixed function is
backward-compatible (same signature), and produces a LARGER (more
correct) amplitude. Any consumer that previously worked with the
small wrong amplitude will work even better with the correct
amplitude (e.g., Math82-Phase-Z continuation will start in a
better-positioned BCC basin). No downstream breakage expected.
Audit of all callers (only \texttt{bcc\_analytic\_seed.py} itself
and Math380 v1's PowerShell here-string) confirms no incompatibility.
