## §5. Devil's advocate (CLAUDE.md §6.3.1)

### Objection α: Math note URL strategy (search vs. direct file reference)

**Claim:** Using GitHub search URLs instead of direct blob URLs is fragile.

**Dismissal:** 
- Math notes are stored at `Docs/math/TECT-MathNN-<descriptor>.tex.txt` with variable naming schemes (e.g., `Math82-AddD` → `TECT-Math82-Addendum-D`)
- Filename mapping requires maintaining a reverse index (expensive)
- Search URL `?q=Math82-AddD` is self-documenting and works for all variants
- User can ctrl+click to see all results and browse related notes (feature, not bug)

**Verdict: DISMISSED** — search strategy is robust and aligns with user workflows.

### Objection β: Abstract truncation loses important context

**Claim:** Limiting abstracts to ~2 sentences / 400 chars loses nuance; readers are directed to PDF anyway, so why abstract at all?

**Rebuttal:**
- Abstracts serve as preview/triage: user can decide whether to download PDF without blind clicking
- 2 sentences = 100+ words, sufficient for stating main result (e.g., "conditional Pillar 1 programme...combining...yields...")
- Full abstracts remain in PDFs; this is a summary layer
- Per `TOE-FACT-SHEET.md`, papers are TECT load-bearing references; abstracts are required due diligence

**Verdict: DISMISSED** — abstracts are appropriately scoped; full text in PDFs.

### Objection γ: 4-category landing organization may not reflect user intent

**Claim:** Users may want to filter by tier (e.g., all T7 papers) or by pillar (all Pillar 4 papers), but category (papers/auxiliary/epochs/top_impact) is orthogonal.

**Rebuttal:**
- Current category organization matches publication workflow: papers = main peer-reviewed, auxiliary = supporting, epochs = intermediate closure, top_impact = high-visibility
- Tier filtering can be added as a secondary UI later (filter buttons on each category page)
- Pillar filtering requires metadata extending each paper entry (feasible but defers to future enhancement)
- Current 4-category structure is stable and published; tier/pillar filters are future work

**Verdict: VALID-with-mitigation** — recommend documenting category semantics in next iteration. Task: Q-2026-05-08-Papers-Tier-Filter-Sidebar (future).
