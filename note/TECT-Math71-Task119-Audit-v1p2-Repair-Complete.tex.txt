\subsection*{\S3 \quad Defect-to-Repair Mapping}

\paragraph{D1 $\to$ R1 (Homogeneous Seed).}
\begin{equation}
  \text{v1.1: } \Psi_{\text{init}} = 0.5 \cdot \mathbf{1}_{3,N,N,N}
  \quad\xrightarrow{\text{v1.2}}\quad
  \Psi_{\text{init}} = \texttt{build\_seed\_bcc}(N, \text{mode}=\text{``minimum''}, \phi_0=0.3)
\label{m71-d1-r1}
\end{equation}
Per Cor. \S2 of Math70, the homogeneous seed is a critical point of the
$k=0$ effective potential and never leaves the $k=0$ subspace under Newton
iteration (proven via Brazovskii shell-bias Fourier-diagonality). The BCC
seed places amplitude on the twelve-star $\{\mathbf{q}_T\}$ (Eq. m70-bcc-seed),
ensuring the iterate samples the non-trivial BCC condensate subspace.

\paragraph{D2 $\to$ R2 (Phase-2 Spectrum).}
The v1.1 code unpacked the third return value of \texttt{newton\_solve} as
\texttt{spectrum\_out} and queried \texttt{spectrum\_out["eigenvalues"]}. The
solver contract (line 1155 of \texttt{tect\_newton\_krylov.py}) returns
$(\Psi, \texttt{history}, \texttt{projector})$ — the Projector instance has
no \texttt{eigenvalues} key. R2 invokes separate Phase-2 helpers:
\begin{equation}
  (\lambda_i, \mathbf{v}_i) \gets \texttt{lanczos\_hessian}(\Psi_\star, \text{params})
  \quad\text{and}\quad
  \Phi_2 \gets \texttt{analyze\_projected\_spectrum}(\lambda_i),
\label{m71-d2-r2}
\end{equation}
where $\Phi_2$ is a \texttt{Phase2Result} dataclass carrying \texttt{eigenvalues},
\texttt{lambda\_min}, \texttt{m\_star\_sq}, etc.

\paragraph{D3 $\to$ R3 (Convergence Check).}
The v1.1 parser \texttt{history[-1].get("converged", False)} was always
\texttt{False} because the per-step history dicts do not carry a global
\texttt{converged} key — the flag is printed but not persisted. R3 implements
the canonical residual check per Math70 Eq. m70-convergence-canonical:
\begin{equation}
  \text{converged} := \|\text{Proj}\,F(\Psi_{\text{sol}}; \text{params})\|_2
  \le \texttt{tol\_newton} \cdot \sqrt{\dim}.
\label{m71-d3-r3}
\end{equation}
This removes the audit's dependence on \texttt{history} metadata and
evaluates the actual residual norm of the returned solution.

\paragraph{D4 $\to$ R4 (Full Config).}
The v1.1 \texttt{params} dict contained only four scalars
$(\mu^2, \lambda, \gamma, \texttt{Newton tol})$. The BCC backend
\texttt{real\_backend\_pt\_bcc\_mixed\_v3.py} requires additional fields:
shell-bias amplitude $\kappa_B$, shell centre $q_0$, shell width $\delta$,
family generators $\{\mathbf{q}_T\}$, box lengths $(L_x, L_y, L_z)$, etc.
All missing fields defaulted to zero, reducing the residual to the
trivial-vacuum energy. R4 substitutes the ad-hoc dict with a call to the
Task \#116 canonical config factory:
\begin{equation}
  \text{params} \gets \texttt{make\_bcc\_config}(N=N, \text{mu2}=\mu^2, \ldots)
\label{m71-d4-r4}
\end{equation}
This is the single source of truth used by the 5-pass v2.6.1 pytest suite,
guaranteeing consistency.

\subsection*{\S4 \quad Exception-Handling Policy (Task \#108 Addendum)}

The v1.1 audit had a single broad handler at line 253:
\begin{verbatim}
  except Exception as e:
      result["error"] = str(e)
\end{verbatim}
This pattern swallows both programming errors (that indicate a code bug and
should propagate with traceback) and legitimate runtime degradation (e.g.,
numerical overflow, CUDA OOM, Lanczos convergence failure).

v1.2 implements Math63 §2A.2 dual dispatch, distinguishing:

\paragraph{Programming Errors (MUST re-raise).}
\begin{equation}
\begin{aligned}
  &\texttt{AttributeError} &&\text{(missing object attribute)} \\
  &\texttt{TypeError} &&\text{(wrong arg type)} \\
  &\texttt{NameError} &&\text{(undefined variable)} \\
  &\texttt{KeyError} &&\text{(missing dict key)}
\end{aligned}
\label{m71-prog-errors}
\end{equation}
These propagate with full traceback, preventing silent data corruption.

\paragraph{Runtime Conditions (log and continue).}
\begin{equation}
\begin{aligned}
  &\texttt{RuntimeError} &&\text{(numerical issue, convergence failure)} \\
  &\texttt{ValueError} &&\text{(invalid computed value)} \\
  &\texttt{MemoryError} &&\text{(OOM)} \\
  &\texttt{np.linalg.LinAlgError} &&\text{(Lanczos non-convergence)}
\end{aligned}
\label{m71-runtime-errors}
\end{equation}
These are logged to stderr with exception type and message; the run continues
to the next grid.

The refactored exception dispatch appears in \texttt{run\_single\_grid} at
lines 376–405. Each degradation mode prints the exception type unconditionally
(no \texttt{if verbose} gate), allowing post-mortem analysis without re-running.
