\section{Technical Details}

\subsection{Standard Pattern Restored}

Each HTML page now follows the canonical structure (matching \texttt{theory.html}, 
\texttt{status.html}, \texttt{toe.html}):

\begin{verbatim}
<header class="site" id="site-header"></header>
<main class="wrap" id="page-content"></main>
<footer class="site" id="site-footer"></footer>

<script src="data/site.js"></script>
<script src="data/<page>.js"></script>
<!-- page-specific scripts -->
<script src="assets/js/tect-render.js"></script>
<script src="assets/js/tect-nav.js"></script>
<script>TECT.render('<page>', window.TECT_<PAGE>);</script>
<!-- optional: post-render helper invocations -->
</verbatim}

The \texttt{TECT.render()} call (line 240 of \texttt{tect-render.js}):
\begin{itemize}
  \item Renders header via \texttt{renderNav(pageId)}
  \item Renders main content from \texttt{pageData.title}, \texttt{.subtitle}, \texttt{.blocks}
  \item Renders footer via \texttt{renderFooter(pageData)}
\end{itemize}

\subsection{Papers Pages Data Structure}

Each papers sub-page now has a simple \texttt{window.TECT\_<CATEGORY>} with:
\begin{itemize}
  \item \texttt{title}: page heading (e.g., ``Auxiliary Papers'')
  \item \texttt{subtitle}: muted description
  \item \texttt{blocks}: single HTML-block container for the PDF grid
\end{itemize}

The rendering pipeline:
\begin{enumerate}
  \item \texttt{TECT.render()} populates header, main, footer
  \item Inline script calls \texttt{TECT.renderPaperGrid('papers-grid', 
    window.TECT\_PAPERS\_PDF\_INDEX, 'category-key')} to populate the grid
  \item CSS (inline in \texttt{<style>}) handles card layout and tier badges
\end{enumerate}

Category keys (from \texttt{papers\_pdf\_index.js}): \texttt{papers}, \texttt{auxiliary}, 
\texttt{epochs}, \texttt{top\_impact}.

\section{Devil's-Advocate Review (CLAUDE.md \S6.3.1)}

\subsection{Objection $\alpha$: Data loss}

\textbf{Claim}: Converting 75 KB hand-curated \texttt{papers.js} to minimal structure 
loses valuable narrative.

\textbf{Status}: \textbf{DISMISSED}

\textbf{Counter-argument}: Old \texttt{papers.js} was never rendered in the HTML; 
it was metadata embedded in the JS file but never displayed on the web. The old 
structure is backed up as \texttt{papers\_narrative.js.bak} and operator can recover 
any necessary content. The new structure delegates page content to 
\texttt{papers\_pdf\_index.js} (the canonical paper registry) and 
\texttt{papers-render.js} helpers, which is the correct separation of concerns.

\subsection{Objection $\beta$: Code page loses functionality}

\textbf{Claim}: Replacing 246-line auto-generated inventory with 4-card navigation 
removes downloadable code listing.

\textbf{Status}: \textbf{VALID} (operator-acknowledged)

\textbf{Mitigation}: Operator explicitly requested this change. The old inventory 
is backed up as \texttt{code\_dump.js.bak}. The new simple navigation is intentional 
pending v3 mirror cutover; downloadable code archives will be re-introduced later 
when \texttt{code/} mirror tree stabilizes.

\subsection{Objection $\gamma$: Rendering brittleness}

\textbf{Claim}: Inline post-render scripts calling \texttt{TECT.renderPaperGrid()} 
after \texttt{TECT.render()} could fail if render completes asynchronously.

\textbf{Status}: \textbf{DISMISSED}

\textbf{Counter-argument}: \texttt{TECT.render()} is synchronous (tect-render.js 
lines 240--277 show direct DOM manipulation, no \texttt{setTimeout}/\texttt{async}). 
The paper grid container \texttt{<div id="papers-grid"></div>} is placed synchronously 
by \texttt{TECT.render()} into the main block; the inline script then immediately 
populates it. Script execution order is guaranteed by HTML parser.
