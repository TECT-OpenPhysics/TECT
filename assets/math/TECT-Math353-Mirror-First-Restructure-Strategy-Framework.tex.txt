\section{Problem statement}

The Math352 commit (2026-05-07) shipped \texttt{migrate\_to\_lowercase\_code.py} v1.1 as the planned mechanism for Phase B-5/6/7 (lowercase \texttt{code/} root). A first execution attempt failed; the rollback was clean and the script was iterated to v1.1 with Path-segment-pattern fixes and a \texttt{PROTECTED\_FILES} guard. Operator audit immediately following identified three structural objections to the lowercase-rename approach.

\paragraph{Objection (a): Local rename is disruption-only.} Renaming \texttt{Codes/} to \texttt{code/} on the canonical tree breaks every in-flight numerical run (\texttt{RunRecorder} JSON paths), every Math-note Markdown citation (over 700 notes), every PowerShell snapshot script invocation, and every IDE bookmark. The benefit is purely ergonomic on the public-facing surface.

\paragraph{Objection (b): \texttt{Github/} and \texttt{Website/} are not duplicates.} They are functionally distinct: \texttt{Website/} is the local data source for the site; \texttt{Github/} is the publish staging area into which \texttt{Website/} is mirrored as \texttt{Github/site/}, alongside curated subsets of \texttt{Codes/} and \texttt{Docs/}. The naming has obscured this layered architecture.

\paragraph{Objection (c): Hard-coded paths are the actual durable problem.} Markdown citation paths, HTML \texttt{href}s, and JS \texttt{data-*} references inside content files are the long-tail issue. Renaming the directory does not solve them; it only relocates the breakage.

The redesign (this Math note) reframes the entire restructure: \emph{do nothing locally; perform every rename and path rewrite at the mirror boundary}.

\section{Design principles (5)}

\begin{enumerate}[leftmargin=2em]
\item \textbf{Single source of truth at local}: \texttt{Codes/}, \texttt{Docs/}, \texttt{Runs/}, \texttt{Website/} remain canonical. Zero local rename. Zero in-flight numerical-work disruption.
\item \textbf{Single mapping table}: all directory renames, exclusions, and content-path rewrites live in \texttt{Codes/config/mirror.json} v2. The mirror tool reads from this; nothing else carries hard-coded path maps.
\item \textbf{Idempotent mirror}: every snapshot reproduces the same \texttt{Github/} tree from the same local input. A single Boolean toggle (\texttt{v0\_compat\_disable\_renames}) reverts to v0 behaviour.
\item \textbf{Path rewriting at the publish boundary}: Markdown links, HTML \texttt{href}, JS data refs are rewritten by the mirror tool at copy time. Local files retain local paths; mirrored files carry mirror paths. Local and public surfaces decouple cleanly.
\item \textbf{GitHub Pages serves \texttt{Github/site/} only}: the publish surface is exactly one directory. \texttt{Website/} is local-only. \texttt{Github/code/} and \texttt{Github/docs/} are sibling reference mirrors but not Pages-served.
\end{enumerate}

\section{Risk register}

\begin{enumerate}[leftmargin=2em]
\item \textbf{False-positive content rewrite}. The regex \texttt{(?<![\textbackslash w/])Codes/} could match a Python string literal \texttt{"Codes/foo"} and rewrite it to \texttt{"code/foo"}, breaking runtime imports. \emph{Mitigation:} (a) the rewrite applies only to extension whitelist (\texttt{.md}, \texttt{.tex.txt}, \texttt{.js}, \texttt{.html}); Python files (\texttt{.py}) are out of scope; (b) dry-run diff before first apply; (c) post-apply \texttt{verify\_website.py} confirms all hrefs resolve to existing mirror files.
\item \textbf{Mirrored citation breakage}. A Math-note cite ``\texttt{Codes/pde/foo.py}'' rewritten to ``\texttt{code/pde/foo.py}'' must resolve in the mirror tree to \texttt{Github/code/pde/foo.py}. \emph{Mitigation:} \texttt{verify\_website.py} extension (B-$\epsilon$) walks every rewritten cite and asserts target-file existence in mirror; failure aborts snapshot.
\item \textbf{In-flight numerical run break}. \emph{Risk: zero, by design.} Local \texttt{Runs/} is untouched; only the mirror copy of \texttt{Runs/} is renamed to \texttt{code/runs/}, and only inside \texttt{Github/}.
\item \textbf{Mirror-tree size growth or leakage}. Mirroring \texttt{Codes/} may expose internal-only files. \emph{Mitigation:} \texttt{exclude\_from\_mirror} list (deprecated subdirs, \texttt{.npy} seeds, \texttt{.npz} run output, \texttt{*.draft.md} narrative).
\item \textbf{Reverse compatibility loss}. External links to \texttt{tect.../Github/Codes/...} break post-rename. \emph{Mitigation:} \texttt{v0\_compat\_disable\_renames} default-true for one snapshot deprecation window; operator confirms external links; flip to false in second commit.
\end{enumerate}

\section{Devil's-advocate self-test (CLAUDE.md \S 6.3)}

\paragraph{Objection $\alpha$.} ``The mirror tool acquires more responsibility (rename + rewrite + exclusion). Effective bug surface increases.''\\
\textbf{Disposition:} VALID-WITH-MITIGATION. Counter: the responsibility was already there but distributed across hard-coded constants in 4+ files (\texttt{generate\_website.py}, \texttt{github\_sync\_curate.py}, \texttt{audit\_website\_freshness.py}, \texttt{verify\_website.py}). Consolidating into a single config + a single tool reduces effective bug surface. Mitigation: every rule in \texttt{mirror.json} has a unit test in \texttt{Codes/tools/test\_github\_sync\_curate.py} v2.

\paragraph{Objection $\beta$.} ``Path rewriting can be subtly wrong (whitespace, code blocks in Markdown, escaped JS string literals). Risk of silent corruption.''\\
\textbf{Disposition:} VALID-WITH-MITIGATION. Mitigation: (a) extension whitelist excludes \texttt{.py} where false-positives are most likely; (b) post-rewrite syntax check via \texttt{node --check} for JS, \texttt{markdown-it} parse for MD; (c) dry-run diff against golden file before first commit; (d) two-snapshot deprecation window before v2 default.

\paragraph{Objection $\gamma$.} ``Two-stage publish (Local $\to$ Github $\to$ Pages) duplicates files in \texttt{Github/} permanently. Wasteful.''\\
\textbf{Disposition:} DISMISSED. \texttt{Github/} is the publish-staging tree; it must exist as a snapshotted artefact for git history and external mirroring. The duplication is the publish boundary, not a pathological copy. Idempotency makes it free in practice.

\section{Quantitative sanity check (CLAUDE.md \S 6.3.4)}

\begin{enumerate}[leftmargin=2em]
\item \textbf{Tree size}: \texttt{Codes/} 176 tracked files; \texttt{Docs/} $\sim$700 Math notes + policy + status; \texttt{Website/} $\sim$50 HTML/JS/assets. Mirrored size $\sim 1.0\times$ source after \texttt{exclude\_from\_mirror} pruning. PASS.
\item \textbf{Path rewrite rule count}: 5 directory renames $\times$ 4 file extensions = 20 effective regex-application contexts. Manageable. PASS.
\item \textbf{Operation count per snapshot}: $\sim$1{,}000 files copied + rewritten on full snapshot; $\sim$10--50 on incremental. Within current snapshot budget ($\sim$60 s). PASS.
\item \textbf{Reversibility}: single Boolean toggle in \texttt{mirror.json}. Two-snapshot deprecation window provides operator-confirmation gate. PASS.
\item \textbf{Disruption to local workflow}: zero. Numerical runs, IDE bookmarks, Math-note citations, snapshot scripts all unchanged. PASS.
\end{enumerate}

\section{Self-adversarial review (CLAUDE.md \S 6.3.5(a))}

\begin{enumerate}[leftmargin=2em]
\item \textbf{Concrete objection 1:} ``This proposal still requires a non-trivial code change (\texttt{github\_sync\_curate.py} v2). Why not stay with v0 1:1 mirror?''\\
\textbf{Disposition:} VALID-WITH-MITIGATION. Counter: v0 mirror does not solve the \texttt{Codes/}/\texttt{code/} naming inconsistency between local (operator-side) and public (researcher-side) surfaces; the inconsistency was the original Phase B-5/6/7 motivation. Mitigation: phased rollout ($\beta$ shippable independently of $\gamma$/$\delta$); v0 toggle preserved for one snapshot cycle.

\item \textbf{Concrete objection 2:} ``The path-rewrite rules might miss a subtle citation form (URL fragment, embedded JS template literal, HTML attribute other than \texttt{href}).''\\
\textbf{Disposition:} VALID-WITH-MITIGATION. Mitigation: post-snapshot \texttt{verify\_website.py} link-check pass walks every \texttt{href}, \texttt{src}, \texttt{data-*}, and Markdown link reference; any miss is caught before \texttt{git push}. Two-snapshot deprecation window covers operator-side validation of edge cases.

\item \textbf{Concrete objection 3:} ``This proposal does not address what happens when an external researcher clones \texttt{Github/} and tries to run \texttt{Github/code/scripts/snapshot.ps1}. Local-only operator scripts may not work in the mirror.''\\
\textbf{Disposition:} DISMISSED-with-clarification. \texttt{Github/} is a read-only mirror for public consumption. Re-run requires the local canonical tree (\texttt{Contents/}). Mitigation: \texttt{Github/code/README.md} carries the banner ``This is a publish mirror. Operator scripts are reference-only; clone the canonical tree to re-run the pipeline.''
\end{enumerate}

\paragraph{Atomic commit sequence.}
\begin{enumerate}[leftmargin=2em]
\item B-$\alpha$/$\beta$: \texttt{mirror.json} v2 + \texttt{github\_sync\_curate.py} v2 + unit tests. \emph{Single commit.}
\item B-$\gamma$: dry-run, diff against golden tag. If clean, proceed; if not, iterate $\beta$ until clean.
\item B-$\delta$/$\epsilon$/$\zeta$/$\eta$/$\theta$: snapshot pipeline + verifier extension + retirements + ROADMAP supersession. \emph{Single commit.}
\item Snapshot run end-to-end (\texttt{snapshot.ps1 -Message "Math353 mirror-first restructure landed"}). Validates whole pipeline.
\end{enumerate}

\paragraph{Two-snapshot deprecation window.}
Before merging v2 mirror permanently, retain \texttt{v0\_compat\_disable\_renames: true} as default for one snapshot cycle. Operator confirms external links still work, internal verification is clean. Then flip to \texttt{false} as second commit.

\paragraph{Rollback.} \texttt{v0\_compat\_disable\_renames: true} restores v0 mirror behaviour. \texttt{Github/} regenerates from local in next snapshot. Local tree untouched throughout.
