\subsection*{\S0 \quad Overview}

Task #54 requires running the μ² continuation solver from the current seed
to the Math56-Addendum Cor. 1 Option B endpoint: $\mu^2_{\text{target}} = 5 \times 10^{-3}$.

The continuation module \texttt{PDE/continuation\_mu2\_v25.py} is currently at
maturity level **SKELETON** (v2.5.7 as of 2026-04-22): it reads configurations,
builds seeds, and prints diagnostic output, but Phase D (the actual Newton-Krylov
solve) contains a PLACEHOLDER that does not invoke the real solver. Task #104
(integrating the real v2.6.0 Newton-Krylov into the continuation driver) is a
multi-turn change requiring explicit user approval and is **not yet implemented**.

Therefore, Task #54 **cannot be executed in the current sandbox** (torch unavailable)
nor completed in production on the user's machine without Task #104 landing first.

\subsection*{\S3 \quad Tentative User-Run Command (Post-Task #104)}

Once Task #104 has integrated the real solver, the command to produce the
continuation endpoint is:

\begin{verbatim}
cd C:\Dev\TECT2\Contents

python PDE\continuation_mu2_v25.py ^
    --config PDE\config_template_brazovskii.json ^
    --N 32 ^
    --mu2-start -1.0 ^
    --mu2-end 0.005 ^
    --mu2-step 0.05 ^
    --output results\continuation_mu2_v25_full ^
    --quiet

# Expected output:
#   results\continuation_mu2_v25_full\MANIFEST.md
#   results\continuation_mu2_v25_endpoint.json (final converged state)
#   results\continuation_mu2_v25_full\continuation_log.txt
\end{verbatim}

\paragraph{Parameter Notes.}
\begin{enumerate}
  \item \texttt{--N 32}: Start at N=32. Continuation at N=64 or N=128 requires
        proportionally more GPU memory (8 GB recommended for N=64).
  \item \texttt{--mu2-start -1.0, --mu2-end 0.005, --mu2-step 0.05}:
        Schedule of 21 points from $\mu^2=-1.0$ to $\mu^2=0.005$.
  \item \texttt{--output}: Output directory for MANIFEST, intermediate saves, logs.
  \item \texttt{--quiet}: Suppress per-step Newton output; final MANIFEST still printed.
\end{enumerate}

\paragraph{Expected Outputs.}
\begin{enumerate}
  \item \textbf{\texttt{results/continuation\_mu2\_v25\_endpoint.json}}
        $\sim 1$–10 MB, containing:
        \begin{verbatim}
{
  "mu2": 0.005,
  "lambda": -0.43,
  "gamma": 1.62,
  "converged": true,
  "newton_steps": ...,
  "free_energy": ...,
  "Psi_star": [...] or {"dtype": "float64", "shape": [...], "seed_recipe": "..."}
}
        \end{verbatim}

  \item \textbf{\texttt{results/continuation\_mu2\_v25\_full/MANIFEST.md}}
        Human-readable summary: entry count, success count, per-point status table.

  \item \textbf{\texttt{results/continuation\_mu2\_v25\_full/continuation\_log.txt}}
        Diagnostic log: per-point Newton trace, Krylov details, stagnation warnings.
\end{enumerate}

\paragraph{Validation Checklist.}
After the run:
\begin{enumerate}
  \item Check exit code: \texttt{\$? -eq 0} iff all points converged.
  \item Check MANIFEST: entry \texttt{n\_converged} should equal total point count.
  \item Check endpoint JSON: \texttt{mu2} field should be \texttt{0.005} and
        \texttt{converged} should be \texttt{true}.
  \item Inspect continuation\_log.txt for any solver warnings or stagnation signals.
\end{enumerate}

\subsection*{\S4 \quad Task #55/#56 Fallback Path (No Endpoint JSON)}

If Task #54 is not executed, Task #55/#56 (the continuum audit via
\texttt{tools/n64\_continuum\_audit.py} v1.2) will still run, using the fallback
$\mu^2 = 5 \times 10^{-3}$ (see Math71 R5).

# 1. Check for results\continuation_mu2_v25_endpoint.json
# 2. NOT FIND IT (fallback triggered)

\paragraph{Verdict on Fallback.}
The fallback $\mu^2 = 5 \times 10^{-3}$ is the Math56-Addendum Cor. 1 Option B
*intended* target (not a guess). Therefore, proceeding without Task #54 is
**not a workaround** — it is the design-specified behaviour when the continuation
has not been executed. The only downside is loss of the path-history (per-point
convergence record, final Psi, Lagrange-multiplier data).
