\section{Addendum A: sandbox-verified repairs of Blockers B1, B5, and Task \#117}
\label{sec:math68-addendum-A}

This addendum records the three sandbox-executable repairs landed on
2026-04-22 in response to Math68 v0.1 \S3 (Blockers B1--B5) and the
companion report
\begin{verbatim}
PS C:\Dev\TECT2\Contents> python Tools/n64_continuum_audit.py
[WARNING] Backend unavailable: No module named 'PDE'
\end{verbatim}
from the user's 2026-04-22 run.  The remaining blockers
$\{$B2$,$B3$,$B4$\}$ require either a torch-backed execution (B2, B3) or
a continuation-driver refactor (B4) and are deferred to the user CPU
parallel queue with exact CLI specs at the end of this addendum.

\subsection{Repair (A.1): Blocker B5 --- test-config \texttt{Lx/Ly/Lz}}
\label{sec:math68-A1-B5}

\paragraph{Root cause.}  Per Prop.~\ref{prop:math68-B5-fix},
\texttt{PDE/real\_backend\_pt\_bcc\_mixed\_v3.\_params\_lengths}
(line~106) reads
\begin{equation}
  (L_x, L_y, L_z) \;=\; \bigl(
      \mathtt{params["Lx"]},\ \mathtt{params["Ly"]},\ \mathtt{params["Lz"]}
    \bigr),
  \label{eq:math68-addA-params-lengths}
\end{equation}
whereas the pre-repair \texttt{tests/test\_v26\_phase\_d.py} assembled
config dicts as \verb|{"L": 1.0, "N": N, ...}|, so every test
exercising the backend raised
\verb|KeyError: 'Lx'| at line 106 \emph{before} any Krylov dispatch
or Hermiticity check could execute.  Three of the five live-pytest
items (\texttt{test\_pcg\_routing\_spd},
\texttt{test\_symmetrised\_jvp\_hermiticity},
\texttt{test\_v24\_regression}) failed or skipped with this exact
signature.

\paragraph{Repair.}  A new \texttt{tests/conftest.py} exports a single
canonical factory
\begin{equation}
  \mathtt{make\_bcc\_config}(L,N;\,\mu^{2},\lambda,\gamma,\dots)
  \;\longmapsto\;
  \bigl\{L_x{=}L,\,L_y{=}L,\,L_z{=}L,\,L{=}L,\,N,\,\mu^{2},\dots\bigr\},
  \label{eq:math68-addA-make-bcc-config}
\end{equation}
which every test in \texttt{test\_v26\_phase\_d.py} now calls in place
of the bare config literal.  The helper writes \emph{all four}
isotropic keys so that any caller reading either the backend-canonical
\texttt{Lx/Ly/Lz} form or the legacy \texttt{L} alias is satisfied.

\paragraph{Sandbox verification.}  Pytest was executed in the sandbox
with stubbed \texttt{torch} and stubbed
\texttt{real\_backend\_pt\_bcc\_mixed\_v3} modules (the sandbox has no
GPU and no torch build); the stubs are transparent to the B5 fix
because the \texttt{KeyError: 'Lx'} failure occurred at the
\emph{Python-level} config unpacking, not at the torch-backed solver.
The post-repair signature is
\begin{equation}
  \mathtt{pytest\ tests/test\_v26\_phase\_d.py\ -v}
  \;\longrightarrow\;
  \begin{cases}
    \texttt{test\_import\_v26}                   & \text{PASSED}\\
    \texttt{test\_symmetrised\_jvp\_hermiticity} & \text{FAILED (B2)}\\
    \texttt{test\_pcg\_routing\_spd}             & \text{SKIPPED}\\
    \texttt{test\_minres\_fallback\_indefinite}  & \text{PASSED}\\
    \texttt{test\_v24\_regression}               & \text{SKIPPED}
  \end{cases}
  \label{eq:math68-addA-pytest-post-B5}
\end{equation}
and the \texttt{KeyError: 'Lx'} error class no longer appears in any
test's stack trace.  The single remaining failure is
\texttt{AttributeError: 'numpy.ndarray' object has no attribute
'requires\_grad\_'} in
\texttt{\_compute\_adjoint\_jacobian\_vec\_v26} --- the exact
signature of Blocker~B2 (Prop.~\ref{prop:math68-B2-fix}), unmasked by
the B5 repair.  On the user's local machine, where torch is available,
the \texttt{requires\_grad\_} call will succeed and the test will
instead probe the graph-connectedness of the numpy-residual wrapper
(still B2, but a different failure mode).

\subsection{Repair (A.2): Blocker B1 --- solver-name API shim}
\label{sec:math68-A2-B1}

\paragraph{Root cause.}  Per Prop.~\ref{prop:math68-B1-fix},
\texttt{continuation\_mu2\_v25.select\_krylov\_solver()} returns names
from the Math63 v1.7 / Math64 v1.1 routing vocabulary
$\{\mathtt{pcg},\mathtt{minres},\mathtt{fgmres}\}$, while
\texttt{newton\_solve()} validates against the older vocabulary
$\{\mathtt{cg},\mathtt{gmres}\}$ and raises \texttt{ValueError} on any
Math63-vocabulary name.

\paragraph{Repair.}  An inline canonicalisation map
\begin{equation}
  \mathcal{M}_{\mathrm{B1}}:\;
  \bigl\{\mathtt{pcg}\mapsto\mathtt{cg},\ \mathtt{fgmres}\mapsto\mathtt{gmres},\ \mathtt{minres}\mapsto\mathtt{gmres}\bigr\}
  \label{eq:math68-addA-alias-map}
\end{equation}
runs immediately on entry to \texttt{newton\_solve}.  If the requested
name is in $\operatorname{dom}\mathcal{M}_{\mathrm{B1}}$, the argument
is silently canonicalised and a stderr warning is emitted so the
downstream solver choice remains auditable.  If the requested name is
outside
$\{\mathtt{cg},\mathtt{gmres}\}\cup\operatorname{dom}\mathcal{M}_{\mathrm{B1}}$,
the validator raises a \texttt{ValueError} whose message lists all
five names.  The permanent fix --- native MINRES dispatch inside
\texttt{newton\_solve} --- remains tracked as Task \#112 sub-item.

\paragraph{Sandbox verification.}  A direct four-case unit test
exercised \texttt{newton\_solve} on a \texttt{(3,4,4,4)} complex128
seed with (a) an unknown name \texttt{'nonsense'} (expected:
\texttt{ValueError}; observed: pass), (b) \texttt{'minres'} (expected:
shim warning, canonicalisation to \texttt{'gmres'}; observed: pass),
(c) \texttt{'pcg'} (expected: shim warning, canonicalisation to
\texttt{'cg'}; observed: pass), (d) \texttt{'fgmres'} (expected: shim
warning, canonicalisation to \texttt{'gmres'}; observed: pass).  Each
of (b)--(d) emitted exactly one stderr line of the form
\begin{verbatim}
[tect_newton_krylov v2.6 B1-shim] krylov_method=<alias> canonicalised
  to <canonical> (Math68 §3 Prop. math68-B1-fix; native MINRES is Task #112)
\end{verbatim}
The test used a torch stub that intercepts the backend import but does
not affect the shim's control flow.

\subsection{Repair (A.3): Task \#117 --- \texttt{n64\_continuum\_audit.py}
bootstrap and import contract}
\label{sec:math68-A3-117}

\paragraph{Root cause.}  Three compounding defects blocked the user's
\texttt{python Tools/n64\_continuum\_audit.py} invocation on
2026-04-22:
\begin{enumerate}[(a)]
  \item missing \texttt{sys.path} bootstrap --- \texttt{PDE/} and
    \texttt{Tools/} were not on \texttt{sys.path} when the script was
    launched from the repository root (analogue of the Task \#101
    / \#110 case-collision fix already applied to
    \texttt{check\_jacobian\_blocks.py});
  \item the import line
    \verb|from PDE.real_backend_pt_bcc_mixed_v3 import RealBackendPtBccMixedV3|
    referenced a non-existent class --- the backend is a
    \emph{functional} module exporting module-level \texttt{residual}
    and \texttt{hessian\_vec}; no \texttt{RealBackendPtBccMixedV3}
    class exists;
  \item \texttt{Psi\_init = np.ones((N,N,N), dtype=np.complex128)}
    used the scalar-Brazovskii convention, whereas the BCC backend
    contract
    (\texttt{PDE/real\_backend\_pt\_bcc\_mixed\_v3.\_shape3}, line 95)
    enforces
    \texttt{Psi.ndim == 4 and Psi.shape[0] == 3}, complex128.
\end{enumerate}

\paragraph{Repair (v1.0 $\to$ v1.1).}  The canonical three-path
bootstrap
\begin{equation}
  (\mathtt{\_THIS\_FILE\_DIR},\ \mathtt{\_PDE\_DIR},\ \mathtt{\_REPO\_ROOT\_DIR})
  \label{eq:math68-addA-bootstrap}
\end{equation}
is inserted between the stdlib imports and the \texttt{torch} guard.
The class-import line is replaced by the bare sibling form
\verb|import real_backend_pt_bcc_mixed_v3 as _backend|, and the
downstream \verb|backend = RealBackendPtBccMixedV3()| instantiation is
replaced by \verb|backend = _backend| (the module itself).  The seed
shape is corrected to $(3,N,N,N)$ complex128 matching the backend
\texttt{\_shape3} contract.  An \texttt{Lx/Ly/Lz} fallback injection
is added inside \texttt{run\_single\_grid} to mirror the Blocker~B5
contract.

\paragraph{Sandbox verification.}  Running
\texttt{python Tools/n64\_continuum\_audit.py --output /tmp/...}
in the sandbox now produces

[WARNING] Backend unavailable: No module named 'torch'
[WARNING] Newton-Krylov v2.6.0 unavailable: No module named 'torch'

where the critical change is that the
