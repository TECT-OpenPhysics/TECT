\section{Why phase splitting}

A monolithic single-commit cleanup would be unsafe for two reasons.
(i) PDE/ hosts the live Task \#54 execution path; renaming or removing
PDE/ in the same commit as the orphan cleanup risks breaking the
in-flight Newton--Krylov continuation. (ii) The PDE/ historical-data
subdirectories ($\sim 12$ subtrees with $\sim 200$ tracked files among
them) require per-folder review to decide between Runs/legacy/,
Runs/historical/, Backup/, or deletion. A reviewer-grade decision on
each subtree cannot be compressed into a single batch operation.

Therefore Phases A+B+C (low-risk, mechanical, byte-equal mirrors and
clearly-orphan files) are executed now. Phases D-F (high-risk, requires
content judgement) are deferred.

\section{Companion executable}

Phase A+B+C is executed by
\texttt{Codes/scripts/repo\_cleanup\_phase2.ps1}. Run from PowerShell
at the repo root:

\begin{verbatim}
cd C:\Dev\TECT2\Contents
Remove-Item .git\HEAD.lock,.git\index.lock -Force -ErrorAction SilentlyContinue
.\Codes\scripts\repo_cleanup_phase2.ps1
\end{verbatim}

The script preserves git history via \texttt{git mv} for relocations
and \texttt{git rm -r} for retirements. It commits at the end with the
maintainer signature.
