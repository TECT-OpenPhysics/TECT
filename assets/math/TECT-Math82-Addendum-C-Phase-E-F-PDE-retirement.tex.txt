\begin{abstract}
This addendum executes Phase E + F of the 2026-04-24 repository
cleanup, completing the \texttt{PDE/} directory retirement initiated
in Phase A+B+C (Math82) and Phase D (Math82-Addendum-A). The cleanup
proceeds in six Phase-E steps (policy docs, .bat scripts, Phase-1
result txt, historical execution data subtrees, deprecated subtree
preservation, untracked content removal) followed by two Phase-F steps
(byte-equal Python and JSON mirror retirement, then \texttt{PDE/}
top-level removal). The Phase Z continuation run launched in parallel
uses \texttt{Codes\textbackslash pde\textbackslash continuation\_mu2\_v25.py}
and is therefore unaffected by this retirement.
\end{abstract}

\subsection{E.4 --- 32 historical-data subtrees $\to$ \texttt{Runs/historical/<name>/}}

For each subtree under \texttt{PDE/}, the script issues either
\texttt{git mv} (if the subtree has tracked content) or
\texttt{Move-Item} (if all content is untracked / gitignored
.\,npy/.\,log artifacts). This preserves history where it exists
and avoids \texttt{git mv} errors on purely untracked subtrees.

\subsection{E.5 --- \texttt{PDE/deprecated/} $\to$ \texttt{Backup/}}

\texttt{PDE/deprecated/} is explicitly un-ignored and contains
$\sim 55$ tracked files (per \texttt{.gitignore} rule
\texttt{!PDE/deprecated/}). It is preserved as an immutable archive
under
\texttt{Backup/pre-PDE-retirement-2026-04-24/PDE\_deprecated/}.

\subsection{E.6 --- Untracked content removal}

\begin{verbatim}
Remove-Item -Recurse -Force PDE/__pycache__
Get-ChildItem PDE/Psi_*.npy | Remove-Item -Force
\end{verbatim}

\section{Phase F execution plan}

\subsection{F.1 --- \texttt{git rm} byte-equal Python and JSON mirrors}

The 38 \texttt{.py} and 4 \texttt{.json} files at the top level of
\texttt{PDE/} are byte-equal with their canonical counterparts in
\texttt{Codes/pde/} (verified Math82-Addendum-A \S 2.1
\texttt{cmp -s}). They are removed from the index via \texttt{git rm};
git history preserves them.

\subsection{F.2 --- Remove now-empty \texttt{PDE/}}

If all preceding steps succeeded, \texttt{PDE/} is empty.
\texttt{Remove-Item -Recurse -Force PDE} removes the directory itself.
