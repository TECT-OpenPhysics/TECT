# WITHDRAWN

This document proposed a retroactive-manifest procedure for pre-
2026-04-15 runs under `schema_version: "1.0-retro"`.

**Status: withdrawn on 2026-04-15.**

Replaced by the records-cutoff doctrine in
`PDE/RECORDS_CUTOFF.md`. No retro manifests are to be written.
Pre-cutoff runs are pre-cutoff and cannot be cited as evidence.

Rationale for withdrawal:

1. Source-file identification by git commit date is ambiguous when
   uncommitted working-tree edits were present at run time.
2. Regime classification from `(λ, γ)` sign is unreliable when the
   run's `config.json` is missing.
3. Reconstructed provenance is not evidence. A forward-confirmation
   run under the current solver is itself sufficient, so the retro
   manifest adds no information.

This stub is preserved (rather than deleted) under the policy
append-only / never-redact rule. For current doctrine see
`PDE/RECORDS_CUTOFF.md`.
